Benefits of inline SVG

  • svg will be cached / gzipped with the page [performance]
  • no required additional http request [performance]
  • keeps svgs in page scope, allowing styling with classes / css [design]
  • allows symbols to inherit text colors / sizes when using supporting css [design]
  • symbol provides accessible text / roles in one place [accessibility]

Additional useage considerations

  • If your icon will be used to convey meaning, the above approach is fine - if the icon is purely decorative (for example, an icon iside of a button with no funtion of its own), best practice is to add aria-hidden="true" to hide it from screenreaders altogether. eg:
<svg aria-hidden="true" class="inline-block">
	<use xlink:href="#icon--thumbs-up"></use>
</svg>
